# APPENDIX_INSIGHT_TYPES.md  
**Insight Types Appendix — Life.OS**  
*Part of the Life.OS Architecture Documentation Suite*

This appendix catalogs every insight type generated across Life.OS. Insights are structured interpretations of user data produced by the Local LLM and the intelligence engines. They power Life Moments, reviews, notifications, graph updates, and cross‑module intelligence.

Insights are stored in `coach_insights` and emitted as timeline events. Each insight type includes its purpose, metadata, triggers, and integration points.

---

# 1. Overview of Insight Architecture

Insights are generated by:

- Emotional Engine  
- Cognitive Engine  
- Habit Engine  
- Productivity Engine  
- Knowledge Engine  
- Context Engine  
- Life.Coach (cross‑module synthesis)

Each insight includes:

| Field | Description |
|-------|-------------|
| id | UUID |
| source_module | journal, notes, tasks, etc. |
| source_id | Entity ID |
| insight_type | emotional, cognitive, etc. |
| content | Insight text |
| metadata | Scores, tags, examples |
| created_at | Timestamp |

---

# 2. Emotional Insights

Emotional insights interpret mood, sentiment, and emotional patterns.

### emotional_shift
A significant change in emotional state.

### emotional_pattern
A recurring emotional trend across entries.

### emotional_intensity_spike
A sudden increase in emotional intensity.

### emotional_granularity_insight
A pattern in how precisely emotions are expressed.

### emotional_weather
Short‑term emotional climate across days/weeks.

### emotional_trigger_detected
A situation, topic, or habit that consistently affects mood.

### emotional_recovery_pattern
How quickly the user returns to baseline after negative emotions.

---

# 3. Cognitive Insights

Cognitive insights interpret thinking patterns, distortions, and identity themes.

### cognitive_distortion_detected
A biased or unhelpful thinking pattern (e.g., catastrophizing).

### cognitive_distortion_resolved
A distortion that appears to be weakening or disappearing.

### cognitive_pattern
A recurring cognitive theme (e.g., self‑criticism).

### identity_theme
A pattern in how the user describes themselves.

### cognitive_reframe_detected
A shift toward a healthier or more balanced perspective.

### cognitive_breakthrough
A major cognitive shift or realization.

---

# 4. Behavioral & Habit Insights

Behavioral insights interpret routines, habits, and consistency.

### habit_detected
A new habit inferred from repeated behavior.

### habit_streak
A consistent pattern of habit performance.

### habit_breakthrough
A habit becoming stable or automatic.

### habit_loop_detected
Cue → routine → reward pattern identified.

### habit_emotion_correlation
A relationship between habits and emotional patterns.

### habit_productivity_correlation
A relationship between habits and productivity.

### behavioral_trigger
A situation that consistently precedes a behavior.

---

# 5. Productivity Insights

Productivity insights interpret task patterns, cycles, and blockers.

### productivity_cycle
A recurring pattern in productivity across days/weeks.

### productivity_blocker
A consistent obstacle to task completion.

### procrastination_pattern
A pattern of delaying tasks.

### energy_cycle
A recurring pattern in energy levels.

### completion_burst
A period of unusually high task completion.

### project_momentum
A measure of project progress over time.

### productivity_breakthrough
A major shift in productivity behavior.

---

# 6. Knowledge Insights

Knowledge insights interpret concepts, topics, and knowledge evolution.

### concept_detected
A new concept extracted from text.

### topic_detected
A new topic cluster identified.

### concept_cluster
A group of related concepts.

### concept_evolution
How a concept changes over time.

### topic_evolution
How a topic cluster evolves.

### knowledge_breakthrough
A major shift in the user’s knowledge landscape.

### knowledge_gap
A missing or underdeveloped concept area.

---

# 7. Cross‑Module Insights

Cross‑module insights combine data from multiple engines.

### emotion_productivity_link
Mood ↔ task completion relationship.

### emotion_habit_link
Mood ↔ habit consistency relationship.

### emotion_knowledge_link
Mood ↔ topics/concepts relationship.

### cognitive_productivity_link
Thinking patterns ↔ productivity.

### cognitive_habit_link
Thinking patterns ↔ habits.

### knowledge_task_link
Concepts ↔ tasks/projects.

### multi_module_pattern
A pattern spanning three or more modules.

---

# 8. Context‑Aware Insights

Context insights interpret the user’s current state.

### time_based_context
Patterns tied to time of day/week.

### emotional_context
Current emotional state influencing suggestions.

### cognitive_context
Current cognitive patterns influencing suggestions.

### productivity_context
Current productivity state influencing tasks.

### environmental_context (future)
Location, weather, or device context.

---

# 9. Life Moment Insights

Life Moments are special insights representing major breakthroughs.

### emotional_breakthrough
A major emotional shift.

### cognitive_shift
A major cognitive realization.

### habit_breakthrough
A major behavioral change.

### productivity_breakthrough
A major productivity milestone.

### knowledge_shift
A major conceptual or topic evolution.

These insights always generate:

- A Life Moment  
- A timeline event  
- A graph node  
- A notification  

---

# 10. Review Insights

Review insights summarize patterns across time.

### daily_summary
Emotional, cognitive, behavioral, productivity summary.

### weekly_trends
Patterns across the week.

### monthly_arcs
Long‑term emotional/cognitive arcs.

### yearly_themes
Identity themes and long‑term patterns.

### life_story_segment
A narrative segment of the user’s life.

---

# 11. Insight Metadata Structures

### Emotional Metadata
```
{ sentiment: number, mood: string, intensity: number, granularity: number }
```

### Cognitive Metadata
```
{ distortions: string[], themes: string[], reframes: string[] }
```

### Habit Metadata
```
{ habit_id: string, streak: number, correlation: number }
```

### Productivity Metadata
```
{ cycle: string, energy: number, procrastination_index: number }
```

### Knowledge Metadata
```
{ concepts: string[], topics: string[], cluster: string }
```
