# APPENDIX_GRAPH_TYPES.md  
**Graph Node & Edge Types Appendix — Life.OS**  
*Part of the Life.OS Architecture Documentation Suite*

This appendix catalogs every node type, edge type, and graph‑related metadata structure used in the Life Graph. The Life Graph is the semantic backbone of Life.OS, connecting entries, notes, tasks, concepts, habits, insights, and Life Moments into a unified knowledge and behavior graph.

---

# 1. Purpose of the Life Graph

The Life Graph enables Life.OS to:

- Represent all entities as interconnected nodes  
- Detect semantic relationships through embeddings  
- Support cross‑module insights  
- Visualize knowledge, habits, emotions, and productivity  
- Track identity evolution and long‑term patterns  
- Power semantic search and clustering  
- Provide context for Life Moments and reviews  

The graph is dynamic, evolving with every entry, note, task, habit, and insight.

---

# 2. Node Data Model

### Table: `life_graph_nodes`
| Field | Type | Description |
|-------|------|-------------|
| id | string | UUID |
| type | string | Node type |
| embedding | number[] | Vector representation |
| metadata | object | Node‑specific metadata |
| created_at | number | Timestamp |

### Node Metadata Structure
```
{ importance: number, cluster: string, neighbors: string[], tags: string[], scores: object }
```

# 3. Node Types

### entry  
Represents a journal entry.

**Metadata includes:**  
- sentiment  
- mood  
- emotional intensity  
- topics  
- cognitive patterns  

---

### note  
Represents a note or idea.

**Metadata includes:**  
- concepts  
- topics  
- checklist items  
- related entities  

---

### task  
Represents a task or project item.

**Metadata includes:**  
- status  
- priority  
- effort  
- energy  
- procrastination index  

---

### concept  
Represents a semantic concept extracted from text.

**Metadata includes:**  
- frequency  
- related topics  
- example sentences  

---

### topic  
Represents a cluster of related concepts.

**Metadata includes:**  
- cluster id  
- concept list  
- evolution timeline  

---

### habit  
Represents a habit or routine.

**Metadata includes:**  
- streak  
- consistency  
- correlations  

---

### insight  
Represents an insight generated by Life.Coach.

**Metadata includes:**  
- insight type  
- scores  
- tags  
- related entities  

---

### life_moment  
Represents a major breakthrough or shift.

**Metadata includes:**  
- moment type  
- intensity  
- related_ids  
- supporting evidence

---

# 5. Edge Types

### mentions  
Entry or note mentions a concept.

**Used for:**  
- semantic linking  
- concept frequency  
- topic evolution  

---

### related_to  
General semantic relationship between two nodes.

**Used for:**  
- cross‑module linking  
- graph neighborhoods  
- semantic search  

---

### derived_from  
One node is derived from another (e.g., insight from entry).

**Used for:**  
- insight provenance  
- Life Moment evidence  
- review generation  

---

### influences  
One node influences another (e.g., habit → mood).

**Used for:**  
- cross‑module insights  
- behavioral correlations  
- emotional forecasting  

---

### similar_to  
Nodes with high embedding similarity.

**Used for:**  
- semantic search  
- clustering  
- topic detection  

---

### part_of  
Node belongs to a larger structure (e.g., task → project).

**Used for:**  
- project graphs  
- hierarchical visualizations  

---

### evolves_into  
Concept or topic evolution over time.

**Used for:**  
- knowledge evolution  
- identity evolution  
- long‑term reviews  

---

# 6. Graph Clustering

Clusters represent groups of related nodes.

### Cluster Types
- concept clusters  
- topic clusters  
- emotional clusters  
- cognitive clusters  
- habit clusters  
- productivity clusters  
- Life Moment clusters  

### Cluster Metadata
```
{ id: string, type: string, nodes: string[], centroid: number[], summary: string }
```

---

# 7. Graph Layout Types

### force_directed  
Physics‑based layout for general exploration.

### radial  
Nodes arranged in concentric circles by type or importance.

### clustered  
Nodes grouped visually by cluster id.

### timeline_aligned  
Nodes positioned along a chronological axis.

# 8. Graph Events

### graph_node_created  
A new node is added.

### graph_node_updated  
Node metadata or embedding updated.

### graph_edge_created  
A new relationship formed.

### graph_edge_updated  
Edge weight or metadata changed.

### graph_cluster_detected  
A new cluster identified.

These events feed the Timeline and support debugging and visualization.
