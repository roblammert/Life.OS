# EMBEDDINGS_AND_SEARCH.md  
**Embeddings & Semantic Search Specification — Life.OS**  
*Part of the Life.OS Architecture Documentation Suite*

Life.OS uses local embeddings and semantic search to power cross‑module intelligence, concept extraction, clustering, and recommendations. All embeddings are generated locally using WebLLM, stored in IndexedDB, and used to unify Journal, Notes, Tasks, and Storage into a single semantic layer.

This document defines the embedding model, vector store schema, search algorithms, clustering logic, and integration with Life.Coach and the Life Graph.

---

# 1. Purpose and Responsibilities

Embeddings and semantic search enable Life.OS to understand meaning, not just keywords.

### Core Responsibilities
- Generate embeddings for all meaningful text and concepts.
- Store embeddings locally for fast, offline search.
- Support semantic search across all modules.
- Enable clustering, topic detection, and similarity scoring.
- Power cross‑module linking and recommendations.
- Feed the Life Graph with semantic relationships.
- Support Life.Coach insight pipelines.

### Why Embeddings?
Embeddings allow Life.OS to:
- Find similar notes or entries even without shared keywords.
- Detect themes and concepts across modules.
- Suggest tasks based on journal content.
- Identify related notes automatically.
- Cluster ideas into meaningful groups.
- Build a semantic Life Graph.

---

# 2. Embedding Model

Life.OS uses a local embedding model provided by WebLLM.

### Requirements
- Runs fully offline  
- Supports 384–1024 dimensional vectors  
- Fast enough for real‑time updates  
- Deterministic output for identical text  

### Embedding Types
- **Document embeddings** — journal entries, notes, tasks  
- **Chunk embeddings** — long entries split into segments  
- **Concept embeddings** — extracted topics, concepts, people  
- **Query embeddings** — search queries  
- **Graph embeddings** — node representations  

---

# 3. Vector Store Schema

Embeddings are stored in IndexedDB.

### `embeddings`
| Field | Type | Description |
|-------|------|-------------|
| id | string | UUID |
| source_module | string | journal, notes, tasks, storage |
| source_id | string | Entity ID |
| vector | float[] | Embedding vector |
| created_at | number | Timestamp |

### Indexes
- `source_module`
- `source_id`
- `created_at`

# 4. Embedding Generation Pipeline

Embeddings are generated automatically when content changes.

### Stage 1 — Preprocessing
- Strip Markdown  
- Normalize whitespace  
- Remove stopwords (optional)  
- Chunk long text  

### Stage 2 — Embedding Generation
- Generate vector(s) using WebLLM  
- Store in `embeddings` table  

### Stage 3 — Graph Update
- Create or update concept nodes  
- Create semantic edges  
- Update similarity clusters  

### Stage 4 — Cache & Index
- Cache vectors in memory for fast search  
- Update semantic index

# 5. Semantic Search

Semantic search is used across all modules.

### 5.1 Query Flow
1. User enters a search query.  
2. Query embedding is generated.  
3. Vector similarity search is performed.  
4. Results are ranked by cosine similarity.  
5. Results are grouped by module.  
6. UI displays combined results.

### 5.2 Similarity Metric
Life.OS uses **cosine similarity**:

\[
\text{similarity}(A, B) = \frac{A \cdot B}{\|A\| \|B\|}
\]

### 5.3 Ranking Factors
- Cosine similarity  
- Recency  
- Entity type weighting  
- User behavior (future)

# 7. Cross‑Module Linking

Embeddings enable automatic linking across modules.

### Examples
- Journal entry → related notes  
- Note → related tasks  
- Task → related journal entries  
- Workbook metric → related entries  

### Linking Rules
- Similarity threshold (e.g., > 0.75)  
- Recency weighting  
- Concept overlap  
- Graph neighborhood

# 8. Integration with Life Graph

Embeddings feed the Life Graph with semantic edges.

### Node Creation
- Concepts  
- Topics  
- People  
- Clusters  

### Edge Creation
- `related_to` edges between similar entities  
- `co_occurs_with` edges for shared concepts  
- `derived_from` edges for task suggestions  

### Edge Weighting
- Cosine similarity  
- Frequency  
- Recency

### Indexing
- Precomputed similarity matrix (optional)
