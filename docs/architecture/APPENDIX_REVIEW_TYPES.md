# APPENDIX_REVIEW_TYPES.md  
**Review Types Appendix — Life.OS**  
*Part of the Life.OS Architecture Documentation Suite*

This appendix catalogs all review types generated by Life.Coach. Reviews are structured, narrative summaries of the user’s emotional, cognitive, behavioral, productivity, and knowledge patterns across a defined period. They help users reflect, understand long‑term trends, and identify meaningful Life Moments.

---

# 1. Purpose of Reviews

Reviews enable Life.OS to:

- Summarize emotional, cognitive, and behavioral patterns  
- Highlight Life Moments and breakthroughs  
- Surface trends across days, weeks, months, and years  
- Provide narrative context for the user’s life  
- Support long‑term personal growth and self‑understanding  
- Feed the Life Timeline and Life Story  

Reviews are generated locally using the Local LLM and stored in `coach_insights` as review‑type insights.

---

# 2. Review Data Model

### Structure
```
{ id: string, period: string, period_start: number, period_end: number, summary: string, emotional_summary: string, cognitive_summary: string, behavioral_summary: string, productivity_summary: string, knowledge_summary: string, life_moments: string[], highlights: object[], metadata: object, created_at: number }
```

### Stored In
- `coach_insights` (as review insights)  
- `timeline_events` (as review events)  

---

# 3. Review Types

Life.OS supports four primary review types:

- Daily  
- Weekly  
- Monthly  
- Yearly  

Each type has its own structure, depth, and intelligence requirements.

---

# 4. Daily Review

### Purpose
Provide a concise summary of the day’s emotional, cognitive, behavioral, and productivity patterns.

### Components
- Mood summary  
- Emotional intensity  
- Cognitive patterns  
- Habit performance  
- Task completion  
- Productivity cycle snapshot  
- Concept/topic activity  
- Life Moments (if any)  
- Suggested focus for tomorrow  

### Metadata Example
```
{ mood_score: 0.4, productivity_score: 0.6, habit_consistency: 0.8, concept_activity: ["focus", "planning"] }
```

### Typical Length
1–3 paragraphs.

---

# 5. Weekly Review

### Purpose
Identify short‑term patterns and trends across the week.

### Components
- Emotional weather  
- Cognitive themes  
- Habit consistency  
- Productivity cycles  
- Project momentum  
- Topic evolution  
- Life Moments  
- Weekly arcs (emotional, cognitive, productivity)  
- Suggested focus for next week  

### Metadata Example
```
{ emotional_trend: "rising", productivity_cycle: "midweek peak", habit_streaks: 3, major_topics: ["identity", "motivation"] }
```

### Typical Length
3–6 paragraphs.

---

# 6. Monthly Review

### Purpose
Reveal medium‑term arcs and deeper patterns.

### Components
- Emotional arcs  
- Cognitive shifts  
- Habit loops  
- Productivity rhythms  
- Project progress  
- Concept/topic evolution  
- Identity themes  
- Life Moments  
- Monthly highlights  
- Suggested focus for next month  

### Metadata Example
```
{ emotional_arc: "recovering", cognitive_shift: "less self‑criticism", habit_loops: ["journaling", "exercise"], topic_evolution: ["growth", "discipline"] }
```

### Typical Length
5–10 paragraphs.

---

# 7. Yearly Review

### Purpose
Provide a long‑term narrative of the user’s emotional, cognitive, behavioral, productivity, and knowledge evolution.

### Components
- Yearly emotional arc  
- Cognitive evolution  
- Habit formation and decay  
- Productivity cycles  
- Major projects and milestones  
- Concept/topic evolution  
- Identity evolution  
- Life Moments (major breakthroughs)  
- Life Story segment  
- Suggested focus for next year  

### Metadata Example
```
{ identity_themes: ["resilience", "growth"], major_life_moments: 12, project_momentum: ["Life.OS", "fitness"], knowledge_evolution: ["systems thinking", "self‑awareness"] }
```

### Typical Length
10–20 paragraphs.

---

# 8. Life Story Segments

Yearly reviews may generate **Life Story segments**, which are long‑form narrative summaries.

### Structure
- Emotional journey  
- Cognitive evolution  
- Behavioral transformation  
- Productivity arcs  
- Knowledge growth  
- Identity themes  
- Major Life Moments  
- Narrative synthesis  

These segments accumulate across years to form a multi‑year Life Story.

---

# 9. Review Generation Pipeline

### Steps
1. Collect data from all modules  
2. Aggregate emotional, cognitive, behavioral, productivity, and knowledge metrics  
3. Identify Life Moments  
4. Detect arcs, cycles, and themes  
5. Generate narrative summary using Local LLM  
6. Store review as insight  
7. Emit timeline event  
8. Trigger notification  

### Intelligence Engines Involved
- Emotional Engine  
- Cognitive Engine  
- Habit Engine  
- Productivity Engine  
- Knowledge Engine  
- Context Engine  
- Life.Coach
