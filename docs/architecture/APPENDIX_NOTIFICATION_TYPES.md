# APPENDIX_NOTIFICATION_TYPES.md  
**Notification Types Appendix — Life.OS**  
*Part of the Life.OS Architecture Documentation Suite*

This appendix catalogs all notification types used across Life.OS. Notifications are generated by modules, intelligence engines, the Context Engine, and Life.Coach. They help users stay aware of insights, habits, tasks, reviews, and Life Moments—without overwhelming them.

Notifications are stored locally, respect quiet hours, and never leave the device unless sync is explicitly enabled.

---

# 1. Purpose of the Notification System

Notifications in Life.OS serve to:

- Surface meaningful insights  
- Highlight Life Moments  
- Remind users of tasks and habits  
- Deliver context‑aware suggestions  
- Provide review summaries  
- Support long‑term awareness and growth  

Notifications must be:

- Helpful  
- Non‑intrusive  
- Context‑aware  
- Private  
- Local‑only  

---

# 2. Notification Data Model

### Table: `notifications`
| Field | Type | Description |
|-------|------|-------------|
| id | string | UUID |
| type | string | Notification type |
| title | string | Short title |
| body | string | Detailed text |
| created_at | number | Timestamp |
| read | boolean | Read/unread |
| snoozed_until | number | Optional timestamp |
| source_module | string | journal, notes, tasks, etc. |
| source_id | string | Entity ID |
| metadata | object | Additional structured data |

### Metadata Structure
```
{ tags: string[], scores: object, related_ids: object }
```

# 11. Notification Priority Levels

### low
- Routine habit reminders  
- Task suggestions  
- Note prompts  

### medium
- Task due  
- Insight generated  
- Productivity suggestions  

### high
- Life Moments  
- Emotional/cognitive breakthroughs  
- Review summaries  
- System errors  

---

# 12. Quiet Hours Behavior

During quiet hours:

- No notifications are delivered  
- Non‑urgent notifications are queued  
- High‑priority notifications are delayed unless user overrides  
- Snoozed notifications respect quiet hours  

Quiet hours default: **10 PM – 7 AM**

---

# 13. Snoozing Behavior

Users can snooze notifications:

- 10 minutes  
- 1 hour  
- 3 hours  
- Until tomorrow  
- Custom time  

Snoozed notifications reappear unless dismissed.

---

# 14. Notification Rendering Rules

### Required elements
- Title  
- Body  
- Timestamp  
- Icon (based on type)  

### Optional elements
- Tags  
- Related entities  
- Preview (entry, note, task)  

### Interaction
- Tap to open source entity  
- Swipe to dismiss  
- Long‑press to snooze
