# KNOWLEDGE_ENGINE.md  
**Knowledge Intelligence Engine — Life.OS**  
*Part of the Life.OS Architecture Documentation Suite*

The Knowledge Engine is responsible for extracting concepts, clustering ideas, building semantic relationships, and maintaining the user’s evolving personal knowledge base. It powers semantic search, cross‑module linking, topic clustering, and the conceptual layer of the Life Graph.

This document defines the knowledge extraction model, clustering logic, concept taxonomy, insight types, data sources, and integration with Life.Coach, the Life Graph, and the Review System.

---

# 1. Purpose and Responsibilities

The Knowledge Engine helps users understand **what they know**, how their ideas evolve, and how concepts relate across their life.

### Core Responsibilities
- Extract concepts and topics from journal entries and notes.
- Cluster related ideas into themes.
- Maintain a semantic knowledge graph.
- Detect emerging topics and fading interests.
- Suggest related notes, tasks, and journal entries.
- Support semantic search and cross‑module linking.
- Contribute to reviews and Life Moments.

### Why Knowledge Intelligence?
Knowledge intelligence helps users:
- See patterns in their thinking  
- Connect ideas across time  
- Strengthen learning and creativity  
- Build a personal knowledge base  
- Understand how their interests evolve  

---

# 2. Knowledge Data Model

Knowledge insights are stored in IndexedDB as part of the `coach_insights` table.

### `coach_insights` (knowledge subset)
| Field | Type | Description |
|-------|------|-------------|
| id | string | UUID |
| source_module | string | journal, notes |
| source_id | string | Entity ID |
| insight_type | string | concept, topic, cluster |
| content | string | Concept or cluster summary |
| metadata | object | Embeddings, related IDs, scores |
| created_at | number | Timestamp |

---

# 3. Concept Extraction

Concept extraction identifies meaningful ideas from text.

### Concept Types
- People  
- Places  
- Projects  
- Topics  
- Skills  
- Interests  
- Values  
- Entities (nouns)  
- Actions (verbs)  

### Extraction Rules
- Use embeddings to identify semantic units.
- Remove stopwords and generic terms.
- Merge similar concepts (e.g., “exercise” and “working out”).
- Track concept frequency and recency.

### Concept Metadata
- Embedding vector  
- Frequency count  
- Recency score  
- Related concepts  
- Related entries/notes  

---

# 4. Topic Detection

Topics are higher‑level themes composed of multiple concepts.

### Topic Formation
A topic is formed when:
- Multiple related concepts cluster together  
- Concepts appear across multiple entries/notes  
- Concepts share semantic similarity  

### Topic Metadata
- Topic label (generated by Life.Coach)  
- Representative concepts  
- Representative entries/notes  
- Strength score  
- Evolution over time  

---

# 9. Integration with Life Graph

Concepts and topics become nodes in the Life Graph.

### Node Types
- **concept**  
- **topic**  
- **cluster**  

### Edges
- concept → entry (mentions)  
- concept → note (mentions)  
- concept ↔ concept (related_to)  
- topic ↔ topic (related_to)  
- concept → task (derived_from)  

### Graph Insights
- Concept clusters  
- Topic evolution  
- Idea relationships
