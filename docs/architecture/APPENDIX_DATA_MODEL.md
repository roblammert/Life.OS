# APPENDIX_DATA_MODEL.md  
**Unified Data Model — Life.OS**  
*Part of the Life.OS Architecture Documentation Suite*

The Unified Data Model defines the core schemas, relationships, identifiers, and metadata structures used across all Life.OS modules and intelligence engines. It ensures consistency, interoperability, and long‑term stability for entries, tasks, habits, insights, embeddings, graph nodes, reviews, and memory records.

## Purpose

The data model provides a single, coherent foundation for storing, retrieving, analyzing, and connecting all user-generated and system-generated data. It ensures that every engine—Emotional, Cognitive, Habit, Productivity, Context, Knowledge, Graph, Memory, Review, and Life Coach—can interoperate seamlessly.

## Core Principles

- **Unified identifiers** — every object has a globally unique ID  
- **Temporal anchoring** — all objects include timestamps  
- **Cross‑module linking** — objects reference each other through stable IDs  
- **Metadata‑rich** — objects include structured metadata for analytics and intelligence  
- **Versioned** — schemas and embeddings include version fields  
- **Extensible** — plugins can add fields without breaking core compatibility  
- **Human‑readable + machine‑optimized** — dual storage formats  

These principles ensure long-term maintainability.

---

## Primary Object Types

Life.OS defines several core object types:

- **Entry** — journal entries, notes, reflections  
- **Task** — tasks, subtasks, completions  
- **Habit** — habits, streaks, consistency  
- **Insight** — emotional, cognitive, habit, productivity, contextual, conceptual  
- **Life Moment** — significant events  
- **Review** — daily, weekly, monthly, yearly  
- **Embedding** — vector representations  
- **Graph Node** — concepts, themes, states, patterns  
- **Graph Edge** — relationships between nodes  
- **Memory Record** — long‑term patterns and identity themes  
- **Context State** — real‑time situational metadata  
- **Automation** — triggers, conditions, actions  
- **Notification** — reminders, insights, prompts  

Each object type includes standardized metadata fields.

---

## Global Metadata Fields

All objects share a common metadata structure:

- **id** — globally unique identifier  
- **type** — object type  
- **created_at** — timestamp  
- **updated_at** — timestamp  
- **source** — user, engine, plugin, automation  
- **tags** — optional labels  
- **visibility** — private, protected, shared, redacted  
- **version** — schema version  

This ensures consistency across modules.
